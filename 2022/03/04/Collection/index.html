Iterator对象称为迭代器，遍历元素（只用于遍历集合，本省并不存放对象），只遍历一次，若需要再次遍历，需要重置迭代器。

增强for循环，就是简化版的迭代器，本质是一样的

Set:无序，没有索引，不允许重复，最多包含一个null，不能使用索引方式遍历

HashSet:实现了Set接口，实际上是HashMap，可以存放null值，但只能有一个，不保证元素是有序的，取决于hash后确定索引结果（不保证存放顺序和取出顺序一致），不能有重复元素

底层机制：

LinkedHashSet:是HashSet的子类，底层LinkedHashMap，底层维护了一个数组+双向链表，根据元素的hashCode值来决定元素的储存位置，同时是用链表维护元素的次序图，使得元素看起来是以插入顺序保存的，不允许添加重复元素。

List:有序，顺序索引，可根据序号取数据

ArrayList:可加入多个null，由数组实现数据储存，基本等同于Vector，但线程是不安全的，多线程不建议使用

底层机制：

Vector:线程同步

Map:具有映射关系的数据：key-Value，key和value可以是任何引用类型的数据，会封装到HashMap$Node对象中，

key不允许重复，value可以重复，

key可以为null, value也可以为null,

注意key为null，只能有一个，value为null，可以为多个，

常用String类作为Map的key，

key和value之间存在单向一对一关系，即通过指定的key总能找到对应的value

key相同时替换

遍历：

第一组

1.先取出所有的key，通过key取得对应的value

Set keyset = map.keySet();

2.增强for或迭代器

第二组

1.把所有的values取出

Collection values = map.values();

2.增强for或迭代器

第三组

1.通过EntrySet来获取k-v

Set entrySet = map.entrySet();// EntrySet<Map.Entry<K, V>>

2.增强for或迭代器

HashTable:K-V,键和值都不能为null，否则会抛出NullPointerException异常，使用
